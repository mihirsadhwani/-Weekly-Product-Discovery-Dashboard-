# Phase 4 — Frontend

## Phase Goal

Build the user-facing Next.js dashboard that reads `output/products.json` (from Phase 3) and displays the weekly product digest in a clean, premium interface.

---

## Design Philosophy

- **Reference:** Clean SaaS dashboard aesthetic — white cards, subtle borders, generous spacing
- **Layout:** Product grid inspired by Product Hunt card layout
- **Feel:** Professional and fast — no heavy gradients, readable typography, smooth transitions
- **Mobile-first:** 1 col → 2 col → 3 col → 4 col responsive grid

---

## How to Run

```bash
# 1. Install dependencies (from this folder)
cd phase4_frontend
npm install

# 2. Make sure products.json exists one level up:
# ../output/products.json   (generated by phase3_final_output)

# 3. Start dev server
npm run dev
# → Opens at http://localhost:3000

# 4. Build for production
npm run build
npm start
```

---

## Data Source

The app reads from `../output/products.json` relative to the Next.js project root.

In production (Vercel), this file must be bundled with the repo. The Phase 5 scheduler commits the updated file after each weekly run, triggering a Vercel redeploy.

---

## Pages

| Route | Description |
|-------|-------------|
| `/` | Homepage: product grid with category tabs, sort, price filter, search |
| `/product/[id]` | Detail page: full analysis, sentiment bars, pros/cons, verdict, Flipkart CTA |

## Components

| Component | Type | Purpose |
|-----------|------|---------|
| `ProductDashboard` | Client | Root client component — manages all filter state |
| `Header` | Client | Sticky top bar with search input and brand |
| `FreshnessBanner` | Client | "Last updated X ago" dismissible strip |
| `CategoryTabs` | Client | Horizontal scrolling pill filters |
| `FilterBar` | Client | Sort dropdown + dual-range price slider |
| `PriceSlider` | Client | Custom dual-thumb range slider |
| `ProductCard` | Server | Card with image, score, price, quote, CTA |
| `RecommendationBadge` | Server | Buy/Wait/Skip gradient pill badge |
| `ProConList` | Server | Two-column pros and cons display |
| `SentimentBreakdown` | Client | Animated sentiment progress bars |
| `ShareButton` | Client | Web Share API + clipboard fallback with toast |
| `BackToTop` | Client | Floating scroll-to-top button |
| `DashboardSkeleton` | Server | Shimmer loading placeholders |
